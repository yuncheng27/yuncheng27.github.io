文件系统目录标准

  • bin/：一般用户可用，启动时会用到的命令，即在文件系统还没有被挂载时也能够使用的命令
  • boot/：存放开机所需的文件，开机时载入开机管理程序(bootloader)，并映象到内存中
    • grub：开机设置相关文件
    • 内核文件(vmlinuz)
  • dev/：放置设备文件，如disk、dvd、floppy、fd0、hdc、stdin等
  • etc/：包含系统特有的Linux系统引导所需的可编辑配置文件，即用于控制程序运行的本地文件。如gconf、yum.conf、hosts等
    • rc.d：用于存放不同运行等级的启动脚本的链接文件
    • X11
  • home/：用户家目录
  • lib/：用于存放程序的动态库和模块文件，包含C编译程序所需要的函数库，是一组二进制文件，如iptables等
  • media/：用于挂载本地磁盘或其它存储设备
    • cdrom
    • floppy
    • U盘
  • mnt/：用于挂载其他临时文件系统
  • opt/：发行版附加的一些软件包(如ubuntu附带的wps)的安装目录，第三方程序安装目录, 如Linux QQ，Linux opera等
  • root/：root用户的家
  • sbin/：这里面很多是只能root用户执行的命令，一些系统进行更新、备份、还原和开关机用到的命令
  • srv/：存放服务进程所需的数据文件(如www网络服务和ftp服务)和一些服务的执行脚本
  • tmp/：存放各种临时文件
  • usr/：用于存储只读用户数据的第二层次；包含绝大多数的(多)用户工具和应用程序
    • bin/：非必要可执行文件(在单用户模式中不需要)；面向所有用户 bin下含有gcc、python等
    • include/：标准包含头文件
    • lib/：/usr/bin和/usr/sbin中二进制文件的库
    • local/：本地数据的第三层次，具体到本台主机
      • bin/
      • etc/
      • include/
      • lib/
      • share/
      • src/
    • share/：体系结构无关(共享)数据
    • sbin/：非必要的系统二进制文件 例如：大量网络服务的守护进程
    • src/：源代码，例如：内核源代码及其头文件。一般的发行版是不会保留源代码的，需要用户自己下载安装
  • var/：变量文件——在正常运行的系统中其内容不断变化的文件，如日志，脱机文件和临时电子邮件文件，有时是一个单独的分区
    • account/
    • cache/
    • lib/
    • lock/
    • log/
    • run/
    • tmp/
    • spool/
    • mail/
  • proc/：存放记录系统文件状态的文件，如meminfo、cpuinfo、devices、partitions、version等
